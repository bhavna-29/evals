    Datasets¶

    Manage named datasets used as evaluation input.

    A dataset is a versioned collection of input–expected-output rows that experiment runs pull from. Supports creating, listing, and uploading items to datasets.

    Access via sdk.datasets on an AISDK instance, or standalone via Evaluate.dataset.

    Example 
    sdk = AISDK(public_key="pk", secret_key="sk")
dataset = sdk.datasets.create(name="eval-set")
print(dataset["id"])

    description Optional[str]

    create_items(dataset_name, items=None, file_url=None, options=None) ¶

    Upload multiple items to a dataset in configurable batches.

    Items can be supplied directly as a list of dicts, or read from a CSV file. The CSV file must have a header row; column names are mapped directly to item fields (input, expectedOutput, context, etc.).

    Parameters:

    Name Type Description Default
    dataset_name str

    The name of the target dataset.

    		 required
    items Optional[List[dict]]

    List of item dicts. Each dict may contain:

    • input (any): LLM input.
    • expectedOutput (any): Expected result.
    • context (any, optional): Additional context.
    • metadata (any, optional): Arbitrary metadata.
    • id (str, optional): Optional stable item ID.
    • sourceTraceId (str, optional): Linked trace ID.
    • sourceObservationId (str, optional): Linked observation ID.
    • status (str, optional): Item status.

    Defaults to None.

    		 None
    file_url Optional[str]

    Local filesystem path to a UTF-8 CSV file containing dataset items. Mutually exclusive with items. Defaults to None.

    		 None
    options Optional[dict]

    Optional configuration dict. Supported key:

    • batchSize (int): Number of items per API call. Defaults to 100.
    		 None

    Returns:

    Type Description
    CreateDatasetItemResponse

    A dict with success (bool) and itemCount (int) fields.

    Raises:

    Type Description
    ValidationError

    If neither items nor file_url is provided.
    ValueError

    If the CSV file cannot be read or parsed.
    AuthenticationError

    If the API key is invalid or expired.
    TestOpsAPIError

    If any batch API request fails.
    Example 
    # Via AISDK instance
sdk = AISDK(public_key="pk", secret_key="sk")
response = sdk.datasets.create_items(
    dataset_name="eval-set",
    items=[
        {"input": "What is AI?", "expectedOutput": "Artificial Intelligence"},
        {"input": "What is ML?", "expectedOutput": "Machine Learning"},
    ],
)
print(response["itemCount"])  # 2

# Standalone (credentials from environment variables)
from browserstack_ai_sdk import Evaluate
response = Evaluate.dataset.create_items(
    dataset_name="eval-set",
    items=[
        {"input": "What is AI?", "expectedOutput": "Artificial Intelligence"},
        {"input": "What is ML?", "expectedOutput": "Machine Learning"},
    ],
)
print(response["itemCount"])  # 2

    list(page=1, limit=50, name=None) ¶

    List datasets with optional name filtering.

    Parameters:

    Name Type Description Default
    page int

    1-based page number. Defaults to 1.

    		 1
    limit int

    Number of datasets per page. Defaults to 50.

    		 50
    name Optional[str]

    Optional substring filter on dataset name. Defaults to None (no filter).

    		 None

    Returns:

    Type Description
    List[DatasetResponse]

    A list of dataset dicts.

    Raises:

    Type Description
    AuthenticationError

    If the API key is invalid or expired.
    TestOpsAPIError

    If the API request fails.
    Example 
    # Via AISDK instance
sdk = AISDK(public_key="pk", secret_key="sk")
datasets = sdk.datasets.list(limit=20)
for ds in datasets:
    print(ds["name"])

# Standalone (credentials from environment variables)
from browserstack_ai_sdk import Evaluate
datasets = Evaluate.dataset.list(limit=20)
for ds in datasets:
    print(ds["name"])
