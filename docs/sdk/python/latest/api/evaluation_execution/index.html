    Evaluation Execution¶

    A lightweight, on-demand API for scoring LLM outputs against quality evaluators — no dataset, experiment, or run configuration required.

    Send a single input–output pair with a list of metric evaluators to the BrowserStack Evals API and receive structured scores back in real time. Ideal for CI pipelines, ad-hoc debugging, and one-off quality spot-checks.

    Access via sdk.evals or standalone via Evaluate.evaluation_execution.

    Example 
    import os
from browserstack_ai_sdk import AISDK

sdk = AISDK(
    public_key=os.environ["AISDK_PUBLIC_KEY"],
    secret_key=os.environ["AISDK_SECRET_KEY"],
)
result = sdk.evals.evaluate({
    "evaluators": [{"metricName": "answer_relevance"}],
    "data": {"input": "What is AI?", "output": "Artificial Intelligence"},
})
print(result["jobId"])

    evaluate ¶

    evaluate(request_data)

    Execute an AI quality evaluation against the BrowserStack Evals API.

    Sends request_data to the /api/public/evals endpoint and returns structured evaluation results. The request is validated locally before any network call is made.

    Parameters:

    Name Type Description Default
    request_data EvaluationExecutionRequest

    A dict with the following keys:

    • evaluators (list, required): List of evaluator configuration dicts. Each dict must contain metricName (str) and the required params.

    • data (dict, required): The data to evaluate. Each data dict supports:

    • input (str, optional): The user query or prompt.

    • output (str, optional): The model's response.
    • context (list[str], optional): Supporting context passages used to generate the response.
    • expectedOutput (str, optional): Ground-truth answer for correctness metrics.
    • retrievedContexts (list[str], optional): Context chunks retrieved by the RAG pipeline (used by RAG evaluators).
    • referenceContexts (list[str], optional): Ground-truth context documents for context recall metrics.

    • rubrics (dict, optional): Custom scoring rubric for rubric-based evaluators.

    • provider (str, optional): LLM provider to use for the evaluation judge model (e.g. 'openai').

    • model (str, optional): Specific judge model name (e.g. 'gpt-4o').
    • concurrency (int, optional): Number of parallel evaluations (1–10). Defaults to the server default.
    		 required

    Returns:

    Type Description
    EvaluationExecutionResponse

    A dict with success (bool), results (list of per-evaluator score objects), jobId (str), and timestamp (ISO-8601 str).

    Raises:

    Type Description
    ValidationError

    If request_data is malformed, evaluators is empty or missing required fields, data is absent, or concurrency is outside the 1–10 range.
    EvaluationExecutionError

    If the API request fails, the response status code is non-2xx, or the response body does not match the expected schema.
    Example 
    import os
from browserstack_ai_sdk import AISDK

# Via AISDK instance
sdk = AISDK(
    public_key=os.environ["AISDK_PUBLIC_KEY"],
    secret_key=os.environ["AISDK_SECRET_KEY"],
)
result = sdk.evals.evaluate({
    "evaluators": [
        {"metricName": "answer_relevance"},
        {"metricName": "faithfulness"},
    ],
    "data": {
        "input": "What is BrowserStack used for?",
        "output": "BrowserStack is a cloud testing platform for web and mobile apps.",
        "context": [
            "BrowserStack provides real device and browser testing in the cloud.",
            "It supports automated and manual testing across 3000+ devices.",
        ],
    },
})
print(result["success"])   # True
for score in result["results"]:
    print(score["metricName"], score["score"])
# answer_relevance  0.95
# faithfulness      0.88

    import os
from browserstack_ai_sdk import Evaluate

# Standalone (credentials from environment variables)
# Set AISDK_PUBLIC_KEY and AISDK_SECRET_KEY in your environment first
result = Evaluate.evaluation_execution.evaluate({
    "evaluators": [
        {"metricName": "answer_relevance"},
        {"metricName": "faithfulness"},
    ],
    "data": {
        "input": "What is BrowserStack used for?",
        "output": "BrowserStack is a cloud testing platform for web and mobile apps.",
        "context": [
            "BrowserStack provides real device and browser testing in the cloud.",
            "It supports automated and manual testing across 3000+ devices.",
        ],
    },
})
print(result["success"])   # True
print(result["jobId"])     # "job_a1b2c3d4"
