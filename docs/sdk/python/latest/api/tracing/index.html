    Tracing¶

    Instrument your LLM application with traces, spans, generation observations, and media attachments. Two approaches are available:

    • Manual tracing — fine-grained control via AISDK instance methods.
    • Auto tracing — zero-code instrumentation via the Observe class.

    Manual Tracing¶

    Create traces, spans, and generation observations using AISDK instance methods. Use sdk.trace() to start a root trace, then nest start_span(), start_generation(), or start_event() inside it.

    Main client for the BrowserStack AI SDK.

    AISDK extends Langfuse to add BrowserStack-specific tracing, evaluation, dataset management, experiment tracking, and prompt management capabilities. It configures OpenTelemetry trace export to the BrowserStack Evals ingestion endpoint and exposes lazy-loaded sub-clients for every product area.

    Example 
    from browserstack_ai_sdk import AISDK

sdk = AISDK(
    public_key="your-public-key",
    secret_key="your-secret-key",
)
span = sdk.start_span("my-step", input="hello")
span.end(output="world")
sdk.flush()
    Note

    TestOps is an alias for AISDK provided for backward compatibility.

    Functions¶

    trace ¶

    trace(name, **kwargs)

    Create a root trace and return a wrapper for attaching child spans.

    A trace represents a single end-to-end request or operation. Use the returned TraceWrapper to call span() and generate() for nested child observations.

    Parameters:

    Name Type Description Default
    name str

    Human-readable name for the trace.

    		 required
    **kwargs

    Additional keyword arguments forwarded to the underlying Langfuse trace (e.g. input, output, metadata, user_id, session_id).

    		 {}

    Returns:

    Type Description

    A TraceWrapper object exposing span() and generate() helpers for child observations.
    Example 
    sdk = AISDK(public_key="pk", secret_key="sk")
t = sdk.trace("rag-pipeline", input={"query": "What is AI?"})

# Span for retrieval
retrieval = t.span("retrieval")
retrieval.end(output={"docs": ["AI is..."]})

# Generation for LLM call
gen = t.span("generation", input="docs + question")
gen.end(output="generated answer")

sdk.flush()

    start_span ¶

    start_span(name, expected_output=None, context=None, parent_span=None, **kwargs)

    Create a span observation, optionally nested under a parent span.

    Extends the base Langfuse start_span to add BrowserStack-specific expected_output and context attributes, which are stored both on the object and as OpenTelemetry span attributes.

    Parameters:

    Name Type Description Default
    name str

    Human-readable name for this span.

    		 required
    expected_output

    Ground-truth value for evaluation. Stored on the span and written as an OTEL attribute. Defaults to None.

    		 None
    context

    Additional context payload attached to the span. Stored and written as an OTEL attribute. Defaults to None.

    		 None
    parent_span

    Parent span to nest this span under. When None the span is attached to the root trace. Defaults to None.

    		 None
    **kwargs

    Extra keyword arguments forwarded to the underlying span API (e.g. input, output, metadata).

    		 {}

    Returns:

    Type Description

    Span observation object with convenience methods start_generation(), create_event(), and start_span() for building nested observations.
    Example 
    sdk = AISDK(public_key="pk", secret_key="sk")
span = sdk.start_span(
    "keyword-extraction",
    input={"text": "Climate change article..."},
    expected_output=["climate", "emissions"],
)
span.end(output=["climate", "environment"])

    start_generation ¶

    start_generation(name, model=None, prompt=None, response=None, expected_output=None, context=None, parent_span=None, **kwargs)

    Create a generation observation for an LLM call.

    Convenience wrapper around start_observation() that maps the familiar prompt/response/model parameters to the Langfuse input/output/model keys. Also stores BrowserStack-specific expected_output and context attributes.

    Parameters:

    Name Type Description Default
    name str

    Human-readable name for this generation step.

    		 required
    model

    LLM model name (e.g. 'gpt-4o'). Defaults to None.

    		 None
    prompt

    Input prompt text or dict. Mapped to input. Defaults to None.

    		 None
    response

    Generated response text. Mapped to output. Defaults to None.

    		 None
    expected_output

    Ground-truth value for evaluation scoring. Defaults to None.

    		 None
    context

    Additional context payload attached to the generation. Defaults to None.

    		 None
    parent_span

    Parent span to nest this generation under. Defaults to None (attaches to root trace).

    		 None
    **kwargs

    Extra keyword arguments forwarded to start_observation().

    		 {}

    Returns:

    Type Description

    Generation observation object with convenience methods attached.
    Example 
    sdk = AISDK(public_key="pk", secret_key="sk")
gen = sdk.start_generation(
    "chat-response",
    model="gpt-4o",
    prompt="What is the capital of France?",
    response="Paris",
    expected_output="Paris is the capital of France",
)
gen.end()
sdk.flush()

    start_event ¶

    start_event(name, expected_output=None, context=None, parent_span=None, **kwargs)

    Create an event observation (a point-in-time span with no duration).

    Events are lightweight observations used to mark discrete moments within a trace — for example a cache miss, a tool call, or a retrieval step that doesn't span time. Like spans they support expected_output and context for evaluation.

    Parameters:

    Name Type Description Default
    name str

    Human-readable name for this event.

    		 required
    expected_output

    Ground-truth value for evaluation scoring. Defaults to None.

    		 None
    context

    Additional context payload attached to the event. Defaults to None.

    		 None
    parent_span

    Parent span to nest this event under. Defaults to None (attaches to root trace).

    		 None
    **kwargs

    Extra keyword arguments forwarded to start_observation() (e.g. input, metadata).

    		 {}

    Returns:

    Type Description

    Event observation object with convenience methods attached.
    Example 
    sdk = AISDK(public_key="pk", secret_key="sk")
span = sdk.start_span("my-step")
event = sdk.start_event(
    "cache-miss",
    parent_span=span,
    input={"key": "prompt-v3"},
)
event.end()
span.end()

    flush()¶

    Call sdk.flush() before your process exits to ensure all buffered spans are uploaded. Inherited from Langfuse — not specific to any one observation type.

    sdk = AISDK(public_key="pk", secret_key="sk")
span = sdk.start_span("my-step", input="hello")
span.end(output="world")
sdk.flush()  # upload everything before exit

    TraceWrapper¶

    Returned by sdk.trace(). Call .span() to create named child spans. The first call creates the root span; subsequent calls create nested children.

    Wrapper for creating a trace with named child spans.

    Returned by sdk.trace(). Call .span() to create named child spans nested under a root trace. First .span() call creates the root; subsequent calls create children.

    Attributes:

    Name Type Description
    trace_id str | None

    The 32-character hex trace ID. None until first .span() call.
    id str | None

    The root span ID. None until first .span() call.
    Example 
    from browserstack_ai_sdk import AISDK

sdk = AISDK(public_key="pk", secret_key="sk")
t = sdk.trace("rag-pipeline", user_id="user-1")

retrieval = t.span("retrieval", input="user query")
retrieval.end(output="doc chunk")

generation = t.span("generation", input="doc + query")
generation.end(output="answer")

sdk.flush()

    Attributes¶

    trace_id ¶

    trace_id

    The trace ID of the root span as a hex string, or None if no span has been created yet.

    id ¶

    id

    The ID of the root span, or None if no span has been created yet.

    Auto Tracing¶

    Zero-code instrumentation. Call Observe.init() once at startup, then every LLM call is captured automatically.

    Functions¶

    init ¶

    init(*args, **kwargs)

    Initialise the BrowserStack AI SDK auto-tracing system.

    Must be called once before any LLM calls are made. Configures the OpenTelemetry tracer provider, attaches the TestOpsSpanProcessor to export spans, and optionally auto-instruments supported LLM libraries (OpenAI, Anthropic, Google Vertex AI) and HTTP frameworks (Flask, FastAPI, Django).

    Parameters:

    Name Type Description Default
    public_key str

    BrowserStack public API key. Can also be set via the AISDK_PUBLIC_KEY environment variable.

    		 required
    secret_key str

    BrowserStack secret API key. Can also be set via the AISDK_SECRET_KEY environment variable.

    		 required
    service_name str

    Name of the instrumented service. Defaults to 'testops-service'.

    		 required
    tracing_enabled bool

    Enable OpenTelemetry tracing. Defaults to True when Observe.init is called.

    		 required
    debug bool

    Enable verbose debug logging. Defaults to False.

    		 required

    Returns:

    Type Description

    The active TestOpsTracing singleton instance, or

    None if initialisation fails.
    Example 
    from browserstack_ai_sdk import Observe

Observe.init(
    public_key="your-public-key",
    secret_key="your-secret-key",
)

    with_trace ¶

    with_trace(name, fn=None, carrier=None)

    Execute a function within a named trace context scope.

    Creates an OpenTelemetry span that acts as the root context for all LLM calls made inside fn. The span is automatically ended when fn returns or raises. Both synchronous and async functions are supported.

    For distributed tracing, pass carrier from an upstream service to automatically link this trace to the upstream trace — inject_remote_carrier() is called internally.

    Parameters:

    Name Type Description Default
    name str

    Name for the trace span (e.g. the function or operation name shown in the BrowserStack dashboard).

    		 required
    fn

    Callable to execute within the trace context. Supports both sync and async functions. Defaults to None.

    		 None
    carrier

    Optional trace context from an upstream service. See inject_remote_carrier() for accepted formats. Defaults to None.

    		 None

    Returns:

    Type Description

    The return value of fn().
    Example 
    from browserstack_ai_sdk import Observe

# Upstream service — start a new trace
def run_pipeline():
    Observe.set_attribute(Observe.TraceAttribute.SESSION_ID, "sess-1")
    result = openai_client.chat.completions.create(
        model="gpt-4o",
        messages=[{"role": "user", "content": "Hello"}],
    )
    return result

Observe.with_trace("run_pipeline", run_pipeline)

# Downstream service — link to upstream trace
def downstream():
    result = openai_client.chat.completions.create(...)
    return result

carrier = {"traceparent": "00-abc...", "baggage": "..."}
Observe.with_trace("downstream", downstream, carrier=carrier)

    set_attribute ¶

    set_attribute(key, value)

    Set an attribute on the current active span.

    Sets the value both as an OpenTelemetry span attribute and in the baggage context so it propagates to all child spans. Use TraceAttribute enum values for standard fields (SESSION_ID, USER_ID, INPUT, OUTPUT) or plain strings for arbitrary metadata.

    Parameters:

    Name Type Description Default
    key

    A TraceAttribute enum member, or a plain string for metadata. String keys are stored under AIOPS_INTERNAL.metadata.<key>.

    		 required
    value

    The value to set. Dicts and lists are JSON-serialised.

    		 required
    Example 
    from browserstack_ai_sdk import Observe

# Set standard fields using the enum
Observe.set_attribute(Observe.TraceAttribute.SESSION_ID, "sess-abc")
Observe.set_attribute(Observe.TraceAttribute.USER_ID, "user-456")

# Set arbitrary metadata using a string key
Observe.set_attribute("environment", "production")

    get_trace_id ¶

    get_trace_id()

    Return the current active trace ID as a hex string.

    Returns:

    Type Description

    The 32-character hex trace ID if a span is currently active,

    or None if no trace is in progress.
    Example 
    from browserstack_ai_sdk import Observe

def my_handler():
    trace_id = Observe.get_trace_id()
    print(trace_id)  # "4bf92f3577b34da6a3ce929d0e0e4736"

    get_carrier ¶

    get_carrier()

    Serialise the current trace context for propagation to downstream services.

    Returns a dict containing traceparent and any baggage entries that can be forwarded to a downstream service as HTTP headers or a message queue payload. Pass the returned dict to inject_remote_carrier() on the receiving side.

    Returns:

    Type Description

    dict with traceparent (str) and any active baggage

    key–value pairs. Returns an empty dict if no trace is active.
    Example 
    from browserstack_ai_sdk import Observe

def upstream_handler():
    Observe.set_attribute(Observe.TraceAttribute.SESSION_ID, "sess-1")
    carrier = Observe.get_carrier()
    # Forward carrier to downstream service
    requests.post("http://service-b/api", json={"carrier": carrier})

    inject_remote_carrier ¶

    inject_remote_carrier(carrier_input)

    Link this trace to an upstream service's trace context.

    Call this from a downstream service (Host 2) to attach incoming spans to the upstream service's trace ID. Accepts carrier data in the W3C traceparent/baggage format.

    Parameters:

    Name Type Description Default
    carrier_input

    The trace context from the upstream service. Accepts:

    • dict — e.g. ``{"traceparent": "...", "baggage": "..."}
    • JSON str — same structure as dict
    • Plain W3C baggage string
    		 required
    Example 
    from browserstack_ai_sdk import Observe

# Downstream service — link to upstream trace
carrier = {"traceparent": "00-abc...", "baggage": "session.id=123"}
Observe.inject_remote_carrier(carrier)

# LLM calls made after this are linked to the upstream trace
result = openai_client.chat.completions.create(...)

    Media Attachments¶

    Media is exported directly from browserstack_ai_sdk. Attach it as a value in any input, output, or metadata dict when creating spans or generations — the SDK uploads the binary data automatically before each trace flush and stores a compact reference string in the trace payload.

    from browserstack_ai_sdk import AISDK, Media

sdk = AISDK(public_key="your-public-key", secret_key="your-secret-key")

span = sdk.start_span(
    "screenshot-analysis",
    input={
        "image": Media(file_path="/tmp/screen.png"),
        "question": "Are there any UI errors visible?",
    },
)
span.end(output={"analysis": "No visible errors found."})
sdk.flush()

    Creating a Media Object¶

    Provide exactly one of the four creation paths:

    Parameter Type Description
    file_path str Path to a local file. MIME type is inferred from the extension. Any file type accepted (image, audio, video, PDF, …).
    content_bytes bytes Raw in-memory binary data. Must also supply content_type.
    base64_data_uri str A complete base64 data URI: data:<mime>;base64,<data>.
    content_type str MIME type string, e.g. "image/png". Required with content_bytes; optional otherwise.

    From a local file¶

    from browserstack_ai_sdk import Media

media = Media(file_path="/screenshots/home.png")        # image
media = Media(file_path="/recordings/user-session.mp3") # audio
media = Media(file_path="/captures/flow.mp4")           # video

    From raw bytes (must supply content_type)¶

    from browserstack_ai_sdk import Media

with open("chart.jpg", "rb") as f:
    raw = f.read()

media = Media(content_bytes=raw, content_type="image/jpeg")

    From a base64 data URI¶

    import base64
from browserstack_ai_sdk import Media

with open("screenshot.png", "rb") as f:
    b64 = base64.b64encode(f.read()).decode()

media = Media(base64_data_uri=f"data:image/png;base64,{b64}")

    Attaching Media to Traces¶

    Manual tracing (AISDK)¶

    from browserstack_ai_sdk import AISDK, Media
import base64

sdk = AISDK(public_key="your-public-key", secret_key="your-secret-key")

# Vision model — attach image
gen = sdk.start_generation(
    "vision-analysis",
    model="gpt-4o",
    input={"image": Media(file_path="/tmp/chart.png"), "prompt": "Summarise this chart."},
)
gen.end(output={"summary": "The chart shows a 15% growth trend."})

# Audio transcription — from file
span = sdk.start_span("transcription", input={"audio": Media(file_path="/tmp/call.mp3")})
span.end(output={"transcript": "Hello, I need help with my order..."})

# From raw bytes
with open("recording.mp3", "rb") as f:
    raw = f.read()
span2 = sdk.start_span("audio", input={"audio": Media(content_bytes=raw, content_type="audio/mpeg")})
span2.end(output={"transcript": "Thank you for calling support."})

# From a base64 data URI
b64 = base64.b64encode(open("chart.png", "rb").read()).decode()
span3 = sdk.start_span("chart", input={"chart": Media(base64_data_uri=f"data:image/png;base64,{b64}")})
span3.end()

sdk.flush()

    Auto tracing (Observe)¶

    from browserstack_ai_sdk import Observe, Media

Observe.init(public_key="your-public-key", secret_key="your-secret-key")
Observe.set_attribute("screenshot", Media(file_path="/tmp/screen.png"))

response = openai_client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "Describe this screenshot."}],
)

    Supported Content Types¶

    Any MIME type is accepted. Common examples:

    Category MIME types
    Images image/png, image/jpeg, image/webp, image/gif, image/svg+xml
    Audio audio/mpeg (mp3), audio/wav, audio/ogg, audio/webm
    Video video/mp4, video/webm, video/ogg
    Documents application/pdf, text/plain, text/html

    TraceAttribute¶

    Use these enum values with Observe.set_attribute() to attach standard metadata to the current span.

    Attribute Value Use for
    TraceAttribute.SESSION_ID session.id Group all traces from a single user session. Pass any unique session identifier (UUID, cookie value, etc.).
    TraceAttribute.USER_ID user.id Associate traces with a specific end-user in your application.
    TraceAttribute.INPUT AIOPS_INTERNAL.trace.input Set the trace-level input shown in the BrowserStack dashboard (e.g. the user's question).
    TraceAttribute.OUTPUT AIOPS_INTERNAL.trace.output Set the trace-level output shown in the BrowserStack dashboard (e.g. the model's final answer). 
    from browserstack_ai_sdk import Observe

Observe.set_attribute(Observe.TraceAttribute.SESSION_ID, "sess-abc123")
Observe.set_attribute(Observe.TraceAttribute.USER_ID, "user-456")
Observe.set_attribute(Observe.TraceAttribute.INPUT, "What is BrowserStack?")
Observe.set_attribute(Observe.TraceAttribute.OUTPUT, "BrowserStack is a cloud testing platform.")
